Kode SQL ini digunakan untuk membangun sebuah basis data sederhana yang menyimpan informasi mengenai bunga Iris, salah satu dataset paling populer dalam bidang machine learning. Bentuk struktur seperti ini lazim digunakan dalam analisis data maupun penerapan machine learning, karena memadukan informasi kategorikal (jenis spesies) dengan data numerik (hasil pengukuran).

3. Menyatukan Database#

import pandas as pd

query_sepal = "SELECT id, class, sepal_length, sepal_width FROM sepal_postgree"

conn_pg.close()

# -----------------------
# Ambil data dari MySQL
# -----------------------
conn_mysql = pymysql.connect(
    host='localhost',
    user='root',
    password='',
    database='petal_mysql'
)

query_petal = "SELECT id, petal_length, petal_width FROM petal"
df_petal = pd.read_sql(query_petal, conn_mysql)
conn_mysql.close()

Kode di atas digunakan untuk mengambil data dari dua sistem basis data yang berbeda, yaitu PostgreSQL dan MySQL, dengan memanfaatkan Python serta library pandas.

Pada bagian pertama, kode membuka koneksi ke PostgreSQL menggunakan library psycopg2. Setelah koneksi berhasil, dijalankan query untuk mengambil kolom id, class, sepal_length, dan sepal_width dari tabel sepal_postgree. Hasil query tersebut dimuat ke dalam sebuah DataFrame pandas bernama df_sepal, sehingga data menjadi lebih mudah diolah maupun dianalisis dengan fungsi-fungsi pandas. Setelah pengambilan data selesai, koneksi ke PostgreSQL ditutup agar tidak membebani sumber daya sistem.

Selanjutnya, pada bagian kedua, kode melakukan proses serupa namun untuk database MySQL dengan bantuan library pymysql. Koneksi diarahkan ke database petal_mysql, kemudian dijalankan query untuk mengambil kolom id, petal_length, dan petal_width dari tabel petal. Data hasil query dimasukkan ke dalam DataFrame pandas bernama df_petal. Setelah data berhasil dimuat, koneksi ke MySQL juga ditutup untuk efisiensi penggunaan sumber daya.

Dengan pendekatan ini, Python dapat mengakses dan mengelola data dari dua sumber basis data berbeda sekaligus. Data dari PostgreSQL (sepal) dan MySQL (petal) bisa dianalisis secara terpisah atau digabungkan sesuai kebutuhan, sehingga mempermudah integrasi dan analisis data lintas platform.

import pandas as pd
from sqlalchemy import create_engine

# PostgreSQL
engine_pg = create_engine("postgresql+psycopg2://postgres:nurilanjar@localhost/postgres")
df_sepal = pd.read_sql("SELECT id, class, sepal_length, sepal_width FROM sepal_postgree", engine_pg)

# MySQL
engine_mysql = create_engine("mysql+pymysql://root:@localhost/petal_mysql")
df_petal = pd.read_sql("SELECT id, petal_length, petal_width FROM petalmysql", engine_mysql)

df_merged = pd.merge(df_sepal, df_petal, on='id', how='inner')

for col in ['sepal_length', 'sepal_width', 'petal_length', 'petal_width']:
    df_merged[col] = df_merged[col].astype(float)

Kode ini digunakan untuk mengambil sekaligus menggabungkan data dari dua database berbeda, yaitu PostgreSQL dan MySQL, dengan bantuan Python menggunakan library pandas dan SQLAlchemy.

Pada bagian pertama, dibuat sebuah engine koneksi ke PostgreSQL melalui fungsi create_engine. Dengan koneksi ini, dijalankan query untuk mengambil kolom id, class, sepal_length, dan sepal_width dari tabel sepal_postgree. Hasil query tersebut dimasukkan ke dalam DataFrame pandas bernama df_sepal.

Kemudian, dibuat engine koneksi lain menuju MySQL. Dari koneksi ini, dijalankan query untuk mengambil kolom id, petal_length, dan petal_width dari tabel petalmysql, lalu hasilnya dimasukkan ke dalam DataFrame df_petal. Dengan cara ini, data dari kedua database dapat diakses dalam format yang sama, yaitu DataFrame pandas.

Setelah kedua DataFrame berhasil dibuat, kode melakukan penggabungan data menggunakan pd.merge dengan basis kolom id dan metode inner join. Hasilnya adalah DataFrame baru bernama df_merged yang hanya berisi baris dengan nilai id yang terdapat pada kedua tabel.

Terakhir, kolom numerik (sepal_length, sepal_width, petal_length, dan petal_width) dikonversi menjadi tipe data float. Konversi ini memastikan bahwa data numerik siap digunakan untuk analisis lebih lanjut, misalnya perhitungan statistik atau pemodelan machine learning.

Dengan pendekatan ini, data yang awalnya tersebar di dua database berbeda (PostgreSQL dan MySQL) dapat digabungkan secara efisien ke dalam satu struktur DataFrame yang konsisten di Python.
