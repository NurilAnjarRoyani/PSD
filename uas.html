Klasifikasi Konsentrasi Alkohol Pada Minuman Keras Menggunakan Time Series Spectrograph#

Business Understanding#

Pendahuluan#

Dataset EthanolConcentration merupakan salah satu dataset multivariate time series yang tersedia dalam UCR/UEA Time Series Classification Repository dan banyak digunakan sebagai benchmark pada penelitian klasifikasi time series. Dataset ini berasal dari pengukuran spektroskopi terhadap sampel whisky dengan beberapa tingkat konsentrasi etanol. Spektrometer merekam bagaimana sampel menyerap, memantulkan, atau mentransmisikan cahaya pada berbagai panjang gelombang, sehingga setiap sampel direpresentasikan dalam bentuk kurva spektrum. Data ini memiliki karakteristik numerik kontinu dan mencerminkan variasi sinyal kimia yang sangat halus antar tingkat konsentrasi etanol, sehingga relevan untuk studi klasifikasi berbasis sinyal.

Secara struktur, dataset EthanolConcentration dibagi menjadi dua bagian, yaitu data training sebanyak 261 sampel dan data testing sebanyak 263 sampel, sehingga total terdapat 524 sampel. Setiap sampel berupa time series multivariat dengan panjang tetap 1751 timestep dan memiliki 3 channel (dimensi). Dengan demikian, satu sampel memiliki ukuran data 1751 × 3 yang merepresentasikan intensitas sinyal spektroskopi pada tiga channel berbeda. Dataset ini dikategorikan sebagai tipe SPECTRO, yang menandakan bahwa data berasal dari sinyal spektroskopi dan bukan dari sensor waktu konvensional seperti accelerometer atau ECG.

Dataset ini memiliki empat kelas target yang merepresentasikan tingkat konsentrasi etanol, yaitu 35%, 38%, 40%, dan 45%. Konsentrasi 40% memiliki makna penting karena merupakan batas minimum kadar alkohol yang diperbolehkan untuk produk Scotch Whisky, sehingga klasifikasi yang akurat pada dataset ini memiliki relevansi langsung terhadap standar dan regulasi industri minuman beralkohol. Perbedaan antar kelas pada data spektrum cenderung sangat tipis dan tidak mudah dibedakan secara visual, sehingga model harus mampu menangkap pola perbedaan yang sangat halus pada sinyal.

Dari sisi penerapan, dataset EthanolConcentration sangat relevan untuk quality control (QC) industri, khususnya dalam industri kimia, farmasi, dan minuman beralkohol. Dengan memanfaatkan teknik machine learning untuk mengklasifikasikan kadar etanol berdasarkan pola time series spektroskopi, proses inspeksi dapat dilakukan secara otomatis, lebih cepat, dan lebih efisien dibandingkan metode laboratorium konvensional. Selain itu, pendekatan ini berpotensi mengurangi biaya operasional, meminimalkan human error, serta mendukung sistem inspeksi real-time, sehingga menjadikan metode komputasi sebagai alternatif pendukung yang kuat dalam proses pengendalian mutu industri modern.

Preprocessing Data#

Dataset EthanolConcentration sudah berbentuk time series numerik dengan panjang tetap (1751 poin) dan 3 channel, sehingga tidak diperlukan perubahan panjang atau segmentasi. Namun, beberapa langkah preprocessing tetap dilakukan, seperti flattening data 3D menjadi 2D agar setiap sampel menjadi vektor fitur tunggal, normalisasi (scaling) menggunakan StandardScaler atau MinMaxScaler untuk menyeimbangkan skala fitur, serta encoding label agar kelas dapat diproses oleh algoritma machine learning. Langkah-langkah ini memastikan dataset siap digunakan untuk pelatihan model klasifikasi.

Normalisasi#

Kode ini digunakan untuk melakukan normalisasi fitur pada dataset EthanolConcentration agar setiap nilai pada semua channel berada dalam rentang yang sama (0 hingga 1). Karena MinMaxScaler dari scikit-learn bekerja pada data 2D (bentuk [samples, features]), pertama-tama data 3D (samples, timesteps, channels) diubah menjadi 2D dengan reshape(-1, X_train.shape[-1]), sehingga semua titik waktu dari setiap channel diperlakukan sebagai fitur tunggal. Setelah dilakukan scaling dengan fit_transform pada data training dan transform pada data testing, data dikembalikan ke bentuk aslinya (samples, timesteps, channels) agar dapat dipakai untuk analisis time series atau pemodelan selanjutnya. Akhirnya, kode mencetak bentuk dataset setelah scaling untuk memastikan dimensi tetap sesuai.

from sklearn.preprocessing import MinMaxScaler

scaler = MinMaxScaler()

# reshape menjadi 2D sebelum scaling: (samples * timesteps, features)
X_train_2d = X_train.reshape(-1, X_train.shape[-1])

# lakukan scaling

# kembalikan ke bentuk semula (samples, timesteps, features)
X_train = X_train_scaled.reshape(X_train.shape)

print("Shape train setelah scaling:", X_train.shape)

Missing Value#

Kode ini digunakan untuk memeriksa keberadaan nilai hilang (missing value / NaN) pada dataset time-series EthanolConcentration. Fungsi cek_missing menerima nama dataset dan array data, kemudian menghitung:

  1. Total elemen (data.size)

  2. Jumlah nilai NaN (np.isnan(data).sum())

  3. Persentase nilai hilang terhadap total elemen

Setelah itu, informasi ini dicetak agar mudah dilihat. Fungsi ini diterapkan pada dataset training dan testing (X_train dan X_test) yang berbentuk 3D (samples, timesteps, channels). Untuk label y_train dan y_test yang berbentuk 1D, kode memeriksa NaN hanya jika tipe data adalah float; jika bukan float (misal string/objek), dianggap aman tanpa nilai hilang. Dengan demikian, kita dapat memastikan bahwa dataset bebas dari missing value sebelum preprocessing atau pemodelan machine learning.

import numpy as np

# ===========================================
# CEK MISSING VALUE PADA DATASET 3D
# ===========================================

def cek_missing(name, data):
    total = np.isnan(data).sum()
    total_elemen = data.size
    persen = (total / total_elemen) * 100

    print(f"=== {name} ===")
    print(f"Total elemen        : {total_elemen}")
    print(f"Jumlah missing (NaN): {total}")
    print(f"Persentase missing  : {persen:.6f}%")
    print()

# Cek pada dataset time-series (3D)
cek_missing("X_train", X_train)
cek_missing("X_test", X_test)

# Untuk label (1D), aman jika bukan float
print("=== y_train ===")
print("Missing value:", np.isnan(y_train).sum() if y_train.dtype.kind == 'f' else 0)
print()

print("=== y_test ===")
print("Missing value:", np.isnan(y_test).sum() if y_test.dtype.kind == 'f' else 0)
print()

Deteksi Outlier#

Kode ini digunakan untuk mendeteksi dan menghapus sampel yang mengandung outlier pada dataset time-series EthanolConcentration. Fungsi detect_and_remove_outliers menerima data 3D (samples, timesteps, channels) sebagai input. Pertama, data di-flatten menjadi 2D agar bisa dihitung Interquartile Range (IQR) per channel, kemudian batas bawah (lower) dan atas (upper) ditentukan dengan rumus IQR (Q1 - 1.5×IQR, Q3 + 1.5×IQR). Selanjutnya, tiap elemen dicek apakah berada di luar batas ini untuk menandai outlier. Setelah itu, kode menentukan sampel mana yang memiliki setidaknya satu outlier di salah satu channel atau timestep, dan menghitung jumlah sampel outlier beserta menampilkan 10 indeks pertama sebagai contoh. Sampel yang mengandung outlier kemudian dihapus, menghasilkan dataset baru cleaned_data. Fungsi ini diterapkan pada X_train dan X_test, sehingga diperoleh dataset training dan testing bersih dari outlier beserta mask boolean yang menunjukkan posisi sampel outlier sebelumnya. Dengan cara ini, dataset menjadi lebih bersih dan siap untuk preprocessing atau pemodelan machine learning selanjutnya.

import numpy as np

# ------------------------------------------------------------
def detect_and_remove_outliers(data_3d):
    n_samples, n_time, n_channels = data_3d.shape

    print("=== DATA SEBELUM OUTLIER DIHAPUS ===")
    print(data_3d.shape)

    # Flatten untuk hitung IQR
    data_2d = data_3d.reshape(-1, n_channels)

    Q1 = np.percentile(data_2d, 25, axis=0)
    Q3 = np.percentile(data_2d, 75, axis=0)
    IQR = Q3 - Q1

    lower = Q1 - 1.5 * IQR
    upper = Q3 + 1.5 * IQR

    # Deteksi outlier per elemen data
    outlier_mask = (data_3d < lower) | (data_3d > upper)

    # Deteksi sampel mana yang memiliki outlier
    sample_has_outlier = np.any(outlier_mask, axis=(1,2))

    # Hitung berapa sample yang outlier
    num_outlier_samples = np.sum(sample_has_outlier)

    print("\nTotal sample dengan outlier :", num_outlier_samples)
    print("Sample index yang outlier   :", np.where(sample_has_outlier)[0][:10],
          "(contoh 10 pertama)")

    # Hapus sample outlier
    cleaned_data = data_3d[~sample_has_outlier]

    print("\n=== DATA SETELAH OUTLIER DIHAPUS ===")
    print(cleaned_data.shape)

    return cleaned_data, sample_has_outlier

            st.error(f"File harus memiliki shape (1751, 3). File anda: {df_input.shape}")
        else:
            st.success("File berhasil dimuat")
            st.dataframe(df_input.head())
            X_input = df_input.values.reshape(1, -1)
            X_input_scaled = scaler.transform(X_input)

elif input_option == "Input Manual":
    st.write("Masukkan data untuk masing-masing channel (dipisahkan koma).")

    ch1 = st.text_area("Channel 1", value=",".join(["0"]*1751), height=150)
    ch2 = st.text_area("Channel 2", value=",".join(["0"]*1751), height=150)
    ch3 = st.text_area("Channel 3", value=",".join(["0"]*1751), height=150)

    if st.button("Prediksi Manual"):
        try:
            ch1_vals = np.array([float(x) for x in ch1.split(",")])
            ch2_vals = np.array([float(x) for x in ch2.split(",")])
            ch3_vals = np.array([float(x) for x in ch3.split(",")])

            if len(ch1_vals) != 1751 or len(ch2_vals) != 1751 or len(ch3_vals) != 1751:
                st.error("Semua channel harus memiliki 1751 poin")
            else:
                df_input = np.stack([ch1_vals, ch2_vals, ch3_vals], axis=1)
                X_input = df_input.reshape(1, -1)
                X_input_scaled = scaler.transform(X_input)
        except:
            st.error("Pastikan format input benar, pisahkan angka dengan koma.")

elif input_option == "Gunakan example.txt":
    if os.path.exists(example_path):
        st.write(f"Memuat data dari {example_path}")
        df_input = pd.read_csv(example_path, header=None)
        if df_input.shape != (1751, 3):
            st.error(f"File example.txt harus shape (1751, 3). Ditemukan: {df_input.shape}")
        else:
            st.success("Data example.txt berhasil dimuat")
            st.dataframe(df_input.head())
            X_input = df_input.values.reshape(1, -1)
            X_input_scaled = scaler.transform(X_input)
    else:
        st.error("File example.txt tidak ditemukan di folder data/")

# ----------------------------------
# 4. Prediksi dan Statistik jika data tersedia
# ----------------------------------
if X_input_scaled is not None:
    # Prediksi kelas
    pred_class = logreg.predict(X_input_scaled)[0]

    # Probabilitas prediksi
    pred_prob = logreg.predict_proba(X_input_scaled)[0]

    st.subheader("Hasil Prediksi")
    st.write(f"Prediksi kelas Ethanol: **{pred_class}**")

    st.subheader("Probabilitas Prediksi per Kelas")
    prob_df = pd.DataFrame([pred_prob], columns=[f"Kelas {c}" for c in logreg.classes_])
    st.dataframe(prob_df)

    # Statistik dasar
    st.subheader("Statistik Dasar Time Series")
    stats_df = pd.DataFrame({
        "Mean": df_input.mean(axis=0),
        "Min": df_input.min(axis=0),
        "Max": df_input.max(axis=0),
        "Std": df_input.std(axis=0)
    }, index=[f"Channel {i+1}" for i in range(df_input.shape[1])])

    st.dataframe(stats_df)

  1. Judul Utama : Prediksi Konsentrasi Ethanol dari Time Series

Judul ini menjelaskan tujuan utama halaman, yaitu melakukan prediksi konsentrasi ethanol berdasarkan data time series tiga channel yang dimasukkan oleh pengguna.

  1. Instruksi Input: Masukkan data time series untuk prediksi kelas Ethanol.

Data harus memiliki panjang 1751 poin per channel dan 3 channel. Instruksi ini memberikan informasi dasar mengenai struktur data yang harus disiapkan oleh pengguna. Model hanya bisa memproses data dengan jumlah poin yang spesifik, yaitu 1751 untuk setiap channel, sehingga format input harus mengikuti ketentuan tersebut.

  1. Pilihan Metode Input

Bagian ini menyediakan tiga opsi cara pengguna memasukkan data ke dalam sistem:

  • Upload CSV

    Pengguna dapat mengunggah file CSV berukuran 1751 × 3, yang berisi 3 channel sinyal time series. Fitur ini cocok untuk pengguna yang sudah menyiapkan data dalam file terstruktur.

  • Input Manual

    Pengguna dapat memasukkan data sinyal secara manual untuk ketiga channel. Disediakan tiga area teks besar: Channel 1, Channel 2, Channel 3.
    Masing-masing harus diisi dengan 1751 nilai yang dipisahkan koma. Format ini memungkinkan pengguna menyalin data langsung dari sumber atau dataset lain.

  • Gunakan example.txt Opsi ini mengisi otomatis input menggunakan file contoh (example.txt) yang sudah tersedia di aplikasi. Fitur ini sangat berguna untuk uji coba, demonstrasi, atau bagi pengguna yang ingin mengetahui format data yang benar.

  1. Area Input:

Pada metode “Input Manual”, tiga kotak teks besar disediakan untuk memuat masing-masing 1751 nilai sinyal per channel. Nilai perlu ditulis dalam format: x1, x2, x3, …, x1751
Hal ini memastikan bahwa setiap channel memiliki dimensi yang sesuai dengan kebutuhan model.

  1. Tombol Aksi: Prediksi Manual

Tombol ini digunakan untuk memulai proses perhitungan model jika pengguna memilih metode input manual. Saat ditekan, aplikasi akan:

  • Memeriksa apakah semua channel memiliki 1751 nilai

  • Mengonversi input menjadi numerik

  • Melakukan normalisasi menggunakan scaler

  • Menghasilkan prediksi kelas ethanol

  1. Hasil Prediksi

Setelah proses prediksi selesai, aplikasi menampilkan hasil dalam beberapa bagian:

  • Prediksi kelas Ethanol

    Menampilkan kelas yang diprediksi model, misalnya: E35, E38, E40, atau E45.

  • Probabilitas Prediksi per Kelas

    Bagian ini menunjukkan probabilitas untuk masing-masing kelas ethanol dalam bentuk tabel. Informasi ini membantu pengguna memahami alasan model memilih suatu kelas tertentu.

  • Statistik Dasar Time Series Aplikasi juga menampilkan informasi statistik untuk tiap channel, seperti: Mean, Min, Max, Std (Standard Deviation).

    Statistik ini memberikan gambaran karakteristik sinyal yang dimasukkan pengguna.
